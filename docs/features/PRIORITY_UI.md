## Implementation Notes

### Visual Elements

1. **Priority Dot Size**: 8-10px diameter
2. **Position**: Between category color bar and checkbox
3. **Spacing**: 8px gap from adjacent elements
4. **Visibility**: Hidden for completed tasks

### Color Codes (from ThemeStore.qml)

```qml
prioHigh:   "#F97066"  // Red
prioMedium: "#FFA726"  // Orange  
prioLow:    "#66BB6A"  // Green
```

### QML Example

```qml
Rectangle {
    width: 8
    height: 8
    radius: 4
    color: {
        const priority = modelData.priority || 0
        if (priority === 2) return colors.prioHigh    // Red
        if (priority === 1) return colors.prioMedium  // Orange
        return colors.prioLow                          // Green
    }
    visible: modelData && !modelData.isDone
}
```

## User Experience

### Sorting Behavior

Tasks are sorted in the following order:
1. Priority (High â†’ Medium â†’ Low)
2. Start time (earlier first)

This ensures:
- Urgent tasks always appear at the top
- Users can quickly identify what needs attention
- The most important work is never missed

### Visual Feedback

- Priority dots provide instant visual feedback
- Color coding follows common conventions (red = urgent)
- Dots disappear when tasks are completed
- Smooth color transitions when priorities change

### Accessibility

- Priority is also exposed as a numeric value (0-2)
- Screen readers can announce priority level
- Color is not the only indicator (position matters too)
